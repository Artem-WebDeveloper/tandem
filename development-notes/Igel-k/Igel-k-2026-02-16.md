# Выбор инфраструктуры и аренда сервера

## Задачи: 
- Подготовить среду для развертывания учебного проекта (Frontend + Backend + Database).
- Установить PostgreSQL и настроить удалённый доступ

## Ход работы:

Проект предполагает размещение всех компонентов на одной машине. Необходимо наличие публичного IP-адреса. Без него невозможно организовать удалённое администрирование и прямой доступ к веб-интерфейсу проекта. Приватная сеть здесь пока не нужна.

Итог: Сервер арендован, доступ по SSH получен. В качестве ОС выбрана Ubuntu 22.04 LTS.

#### Установка PostgreSQL и настройка удаленного доступа
Нужно развернуть базу данных и подключиться к ней через IDE (буду использовать DBeaver) с локального компьютера.

Установила PostgreSQL (версия 16) через менеджер пакетов apt и через суперпользователя postgres в оболочке psql сделала тестовую базу данных.

При попытке подключиться через DBeaver вылезала ошибка Connection refused.

Было выяснено, что по умолчанию PostgreSQL слушает только localhost (127.0.0.1). Я отредактировала конфиг ```/etc/postgresql/16/main/postgresql.conf``` и изменила параметр ```listen_addresses``` на '*'. Также настроила аутентификацию в ```pg_hba.conf``` добавив правило ```host all all 0.0.0.0/0 scram-sha-256```, разрешающее вход по паролю из внешней сети.

Итог: После перезагрузки сервиса (```systemctl restart postgresql```) удалось выполнить удалённое подключение.
