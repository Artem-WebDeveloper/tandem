# Реализация регистрации и системы аутентификации с логированием

## Задачи:
- Разработать механизм регистрации новых пользователей.
- Настроить систему аутентификации на основе JWT токенов.
- Создать модель для отслеживания истории входов.
- Переопределить логику выдачи токена для автоматической записи события входа.

## Ход работы:

#### Разработка функционала регистрации

Для создания пользователей я реализовала сериализатор, который валидирует входящие данные и хеширует пароль. В представлении разрешила доступ всем пользователям. Это необходимо для того, чтобы любой мог зарегистрироваться. Такие же настройки будут для авторизации. Для всего остального функционала в дальнейшем будет требоваться авторизация.

#### Настройка токенов и безопасности

Для аутентификации была выбрана работа через токены. При успешном логине клиент получает два уникальных токена ```access``` и ```refresh```. Токен ```access``` используется для идентификации в запросах и живёт достаточно короткое время. Токен ```refresh``` живёт дольше и используется для обновления токена ```access```. Это необходимо для того, чтобы пользователю не надо было слишком часто авторизоваться заново. На фронтенде надо будет предусмотреть запрос обновления через ```refresh``` токен, чтобы не получать регулярно от сервера ответ ```401 Unauthorized```.

#### Модель истории входов и кастомизация логина

Чтобы понимать, когда и кто заходил в систему, я добавила новую модель ```LoginHistory```. Она связывается с пользователем через внешний ключ ```ForeignKey``` и хранит временную метку события. Однако стандартный механизм выдачи токенов просто возвращает ключ. Поэтому необходимо переписать логику стандартного представления для получения токена, чтобы при успешной проверке создавалась запись в таблице истории и генерировались токены. Таким образом все входы в систему будут записаны.

Итог: Настроена система регистрации и авторизации. Добавлен журнал захода пользователей.
