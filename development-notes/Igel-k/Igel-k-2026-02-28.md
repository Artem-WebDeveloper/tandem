# Авторизация и настройка CORS

## Задачи:
- Разработать интерфейс страницы входа (Login Page).
- Реализовать логику взаимодействия с API для получения JWT-токенов.
- Настроить глобальное хранилище для управления состоянием авторизации.
- Настроить бэкенд для кросс-доменных запросов.

## Ход работы:

#### Создание интерфейса и логики запросов
Я разработала страницу авторизации, используя библиотеку компонентов ```Material UI```. Для взаимодействия с бэкендом была написана функция-обработчик на базе ```axios```. Она отправляет учетные данные пользователя на созданный ранее эндпоинт и ожидает в ответ пару токенов ```access``` и ```refresh```.

#### Управление состоянием с помощью Zustand
Чтобы токены были доступны во всем приложении, я создала ```auth.store.ts```, который отвечает за сохранение токенов после успешного входа, очистку данных при выходе и хранение статуса авторизации.

#### Решение проблемы CORS
При первой попытке связать фронтенд с бэкендом в браузере возникла ошибка ```CORS``` (```Cross-Origin Resource Sharing```). Это стандартная защита, которая блокирует запросы, если фронтенд и бэкенд находятся в разных источниках. При этом источники считаются разными даже если хост один, но разные порты. Поэтому я установила дополнительно пакет ```django-cors-headers``` и в ```settings.py``` добавила список разрешённых адресов. Сейчас это только ```http://localhost:5173``` и ```http://127.0.0.1:5173```. Далее будут добавлены адреса, которые принадлежат фронтенду.

Итог: Теперь фронтенд успешно общается с сервером. Пользователь может войти в систему, а его токены сохраняются в приложении.
