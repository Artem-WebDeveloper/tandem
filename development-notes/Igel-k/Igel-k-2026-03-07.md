# Полноценный запуск сервера и все, все, все

## Задачи:
- Запустить бэкенд для проекта на облачном сервере.
- Разобраться с утройством ```Nginx``` и ```Gunicorn```.
- Настроить ```.env``` для бэкенда.
- Создать домен для работы с ```HTTPS```.
- Связать бэкенд и фронтенд в одно целое.

## Ход работы:

#### Запуск бэкенд для проекта на облачном сервере.
Когда уже стало можно регистрировать пользователей и выдавать им токены при авторизации, настало время развернуть всё это на облачном сервере, который был арендован ранее и простаивал без дела уже две недели. Для этого я настроила соединение по ssh для загрузки проекта с ```github```, синхронизировала все пакетные зависимости и подтянула сам код. Далее были проведены миграции и собрана статика. 

#### Настройка Nginx и Gunicorn

Поскольку встроенный сервер ```Django``` предназначен только для разработки, надо переходить на что-то более серъёзное. Я решила использовать классическую связку ```Gunicorn``` и ```Nginx```. ```Gunicorn``` запускает несколько рабочих процессов для обработки запросов. Я настроила его запуск через ```systemd```, чтобы сервер автоматически поднимался после перезагрузки. ```Nginx``` принимает входящий трафик на 80-м порту и перенаправляет его на ```Gunicorn```. Также ```Nginx``` занимается раздачей статических файлов, это нужно для подгрузки некоторых стилей, например, чтобы админка выглядела приятно глазу.

#### Настройка .env для бэкенда

Для безопасности проекта я вынесла все чувствительные данные из settings.py в файл .env. В него переехали:
* Секретный ключ приложения.
* Параметры подключения к базе данных (хост, имя, пароль).
* Флаг ```DEBUG```, который на сервере всегда ```False```.
* Список ```ALLOWED_HOSTS``` и ```CORS_ALLOWED_ORIGINS```.

#### Создание домена для работы с HTTPS

Вообще я не думала, что придётся покупать и регистрировать домен. Достаточно того, что я уже потратилась на сервер. Но домен был необходим для получения SSL-сертификата, который в свою очередь необходим для возможности принимать запросы с ```HTTPS```, ведь ```gihub pages``` требует именно защищённое соединение. Поэтому был выбран бесплатный вариант с ```duckdns.org```. Там я сделала поддомен и указала ```IP``` сервера для перенаправления. Теперь можно добывать сертификат, теперь можно получать запросы с ```gihub pages```.

#### Связка бэкенда и фронтенда в одно целое

После всего проделанного это завершающий этап, который покажет, что все страдания были не зря. Так и вышло. Я обновила ```URL``` адрес в ```.env``` и стала смотреть работают ли запросы на ```duckdns.org```. С первого раза они работать не стали. Зато стали работать со второго, когда я вспомнила, что теперь надо указывать не ```http```, а ```https```.

Итог: всё работает и пока ничего не сломалось.
