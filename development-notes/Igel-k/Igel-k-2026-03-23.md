# Проверка и результаты

## Задачи:

* Создать модель для хранения результатов прохождения квизов
* Разработать систему проверки ответов на квизы
* Исправить мелкие замечания

## Ход работы:

#### Модель результатов

Здесь я пока решила не усложнять. Записываем только номер квиза, пользователя, прошедшего квиз, результат прохождения квиза в процентах и дата прохождения квиза. Если нужно будет что-то ещё, то добавлю позже

#### Обработка результатов

Теперь работает проверка введённых ответов. Сервер получает ```POST``` запрос, где содержится только список ответов и id каждого вопроса. ID пользователя передавать не нужно, так как для этого запроса бэкенд требует авторизацию и именно из неё можно получить все данные о пользователе. ID квиза также не нужно передавать, так как эта информация содержится в url запроса. После проверки, возвращается ответ, который содержит детальную информацию о правильных и неправильных ответах, а также результат прохождения в процентах, общее число правильных ответов и общее число вопросов. Возможно, в будущем добавлю формулировки вопросов.

#### Мелкие исправления

Для удобства работы фронтенда я добавила информацию об общем количестве страниц в выдаче списка всех квизов. Также я добавила в общую информацию о квизе количество вопросов в каждом из них.

**Итог:** Теперь работает проверка ответов и отдаётся детальная информация о результатах прохождения квиза.
