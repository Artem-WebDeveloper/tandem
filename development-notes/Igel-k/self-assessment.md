# Описание работы

В рамках проекта я занималась настройкой и разработкой бэкенда всего приложения, а также участвовала в создании страницы авторизации и регистрации пользователей.

## Основные задачи и вклад

### 1. Сложный бэкенд-сервис: Пользователи и их регистрация и авторизация

Я полностью спроектировала и реализовала модели пользователей, их сериализацию и представление для API:

- продумала необходимые поля для моделей;
- добавила возможность регистрации с ограничениями на пароли:
  - не допускается повторение имени пользователя в пароле;
  - не допускаются только числа в пароле;
  - не допускаются слишком короткие пароли (менее 8 символов);
  - не допускаются слишком распространённые пароли;
- добавила авторизацию с использованием JWT токенов.

### 2. Сложный бэкенд-сервис: Модели квизов и их представления

Я полностью спроектировала и реализовала модели квизов, организовала эндпоинты и обработку результатов:

- создала универсальную модель квиза;
- продумала необходимые поля для всех типов квизов;
- реализовала фильтрацию по необходимым полям;
- добавила пагинацию на список всех доступных квизов;
- реализовала универсальный эндпоинт для добавления/удаления квизов в избранное;
- сделала обработку результатов со статистикой по пройденному квизу;
- доавила эндпоинт для дашборда.

### 3. Custom Auth: Собственная авторизация (JWT)

Добавила авторизацию через jwt токены. Для этого использовалась библиотека `Django rest framework simplejwt`

### 4. Страница авторизации и регистрации

- сделала базовую страницу авторизации пользователей;
- сделала модальное окно для регистрации;
- использовала компоненты Material UI;
- адаптировала компонент под мобильные устройства;
- добавила поддержку выбранной темы.

### 5. Разработка своего бэкенда с локальной базой данных

- арендовала сервер на сайте Selectel;
- настроила базу данных PostgreSQL;
- настроила виртуальное окружение для приложения на Django;
- настроила gunicorn и nginx;
- настроила работу с https;
- интегрировала базу данных в приложение Django;
- продумала организацию хранения данных.

## Основные сложности

- настройка работы с https;
- неочевидные моменты при создании кастомной модели пользователей;
- внедрение дополнительного функционала к уже существующему;
- настройка nginx и gunicorn.
