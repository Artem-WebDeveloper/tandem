## Ход работы

Вспомнил, что мы обсуждали процесс проверки ответов пользователя на недавнем созвоне. По ходу обсуждения были выдвинуты различные варианты проверки ответов, поэтому решил уточнить этот момент сейчас еще раз.  
По итогу было решено проверять ответы на бекэнде, потому что это самый правильный подход, так как он изолирует ответы от пользователя. Для этого нужно немного изменить JSON-схему с вопросами (удалить поле "answer" в вопросах). Также решил добавить вопросам "id", чтобы их можно было перемешивать.

Финальный (надеюсь 😅) вид схемы с вопросами:

```json
{
  "id": "cc-001",
  "section": "Core JS",
  "type": "Code Completion",
  "difficulty": 2,
  "timeLimit": 180,
  "tags": ["array-methods"],
  "version": 1,
  "questions": [
    {
      "id": 1
      "code": "const result = arr.___(x => x > 0);",
      "blanks": ["___"],
      "hint": "This method returns a new array with elements that pass the test"
    }
    ...
  ]
}
```

Схема с ответами пользователя:

```json
{
  "taskId": "cc-001",
  "userAnswers": [

    {
      "questionId": 2,
      "payload": "map"
    }
  ]
}
```
