## Ход работы

Создал общий компонент навигации с 2 кнопками "Назад" и "Следующий вопрос". Сначала хотел хранить управление в квизе, прокидывая лишь колбеки в навигацию, но потом понял, что это не удобно. В итоге перенес все в сам компонент навигации, а через props прокинул номер текущего вопроса, его сеттер и общее число вопросов. Позже, как разберемся с тем, как завершать квиз надо будет добавить и колбек для этого.

К квизу привязалось все нормально, но вот с самим квизом не все так просто. Мне нужен инпут для ввода ответа, который будет сохранять ответ, чтобы при навигации данные пользователем ответы появлялись в инпуте. Также надо где-то хранить все ответы за данный квиз.
Тут возникла проблема:

```
CodeCompletionWidget contains an input of type text with both value and defaultValue props. Input elements must be either controlled or uncontrolled (specify either the value prop, or the defaultValue prop, but not both). Decide between using a controlled or uncontrolled input element and remove one of these props.
```

Нельзя и использовать инпут для ввода значения и заполнять его значением по умолчанию.  
Убрал state для отдельного ответа и в value инпута начал ссылаться на значение из массива всех ответов. По началу была проблема с тем, что все ответы изначально undefined и при смене на пустую строку input переставал был uncontrolled, на что ругался React. Решил это, написав (вроде как, нормальная практика делать именно так):

```typescript
<input
  type="text"
  value={answers[currentQuestionNumber] ?? ''}
  onChange={handleAnswerInput}
/>
```

Все заработало как надо. Создал глобальный state для своего квиза и разделил все части виджета по своим файлам, чтобы оптимизировать перерисовку.

Столкнулся с проблемой переноса текста, посреди которого есть input. Оказалось, что дело не в наличии input, а просто в том, что я поместил все в тег `<pre>`, поэтому исправилось все стандартными строками css:

```css
white-space: pre-wrap;
word-break: break-word;
```

Всё, закончил! Я доволен результатом. Осталось настроить отправку и проверку ответов с отображением результатов, но это уже будет позже, когда обо всем договоримся и будет готов бекэнд.
