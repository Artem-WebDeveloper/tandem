## Ход работы

Начал с изучения документации по Vitest. Не особо в ней много информации о том, какие есть методы для тестирования и как ими пользоваться. Придется разбираться по-другому. В документации testing-library побольше полезной информации. Буду изучать ее.

Думаю, для начала написать пару тестов для общих компонентов QuizProgressBar и QuizNavigation, чтобы разобраться, так как пока не очень понятно, как и какие тесты писать для более сложных компонентов.

Для QuizProgressBar добавил проверку рендера без ошибок и проверку рендера с правильными значениями:

```typescript
describe('QuizProgressBar', () => {
  it('renders without errors', () => {
    render(<QuizProgressBar currentQuestionNumber={1} questionsCount={1} />);
  });

  it('renders with correct values', () => {
    render(<QuizProgressBar currentQuestionNumber={2} questionsCount={5} />);

    const currentQuestion = screen.getByText('Вопрос 2 из 5');
    const progress = screen.getByText('40%');

    expect(currentQuestion).toBeInTheDocument();
    expect(progress).toBeInTheDocument();
  });
});
```

Написал тесты для QuizNavigation. Получилось аж 10. Решил узнать, как можно сделать их компактнее, так как рендер компонента в каждом из них занимает по 8 строчек, при этом не сильно отличаясь от теста к тесту. Изучив вопрос, узнал про практику использования функции setup, которая подготавливает начальные данные для тестов. В нее как раз можно вынести рендер компонента. Также обновил тесты для QuizProgressBar, вынеся общее в setup.
