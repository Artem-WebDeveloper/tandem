# Описание работы

В рамках проекта я занимался разработкой интерактивных квизов и пользовательского интерфейса для их прохождения, а также участвовал в развитии архитектуры приложения и командной разработке.

## Основные задачи и вклад

### 1. Разработка Code Completion Widget (с нуля)

Я полностью спроектировал и реализовал компонент квиза формата заполнения пропусков в коде:

- продумал UX и структуру интерфейса (навигация, прогресс, зона задания);
- разработал JSON-схему квиза, включая поддержку:
  - нескольких вопросов;
  - тегов и сложности;
  - лимита времени;
- реализовал:
  - навигацию между вопросами;
  - хранение и восстановление ответов пользователя;
  - controlled input для корректной работы React;
  - переработал формат ответов под требования бэкенда.

Также участвовал в обсуждении общих компонентов (результаты, теги, layout квиза).

### 2. Управление состоянием (Zustand)

Освоил и внедрил Zustand для хранения состояния квизов:

- реализовал глобальный store для ответов;
- обеспечил синхронизацию UI и состояния;
- исправил ошибки мутации (работа без Immer);
- добавил сброс состояния при выходе из квиза.

### 3. Разработка Async Sorter Widget (Drag & Drop)

Создал второй тип квиза — сортировка элементов с помощью drag-and-drop:

- выбрал и изучил библиотеку dnd-kit;
- реализовал:
  - drag-and-drop между зонами;
  - корректную обработку событий;
  - синхронизацию с Zustand;
- решил проблемы:
  - типизации событий;
  - коллизий (использование closestCorners);
  - багов перемещения элементов.

Дополнительно:

- добавил подсветку синтаксиса через react-syntax-highlighter;
- адаптировал UI под разные темы.

### 4. UI и дизайн (Material UI)

Активно работал с Material UI:

- сопоставил дизайн макета с компонентами MUI;
- реализовал:
  - прогрессбар (LinearProgress);
  - скелетоны загрузки;
  - обработку ошибок (Alert);
- настроил темизацию:
  - темная и светлая темы;
  - типографика;
  - динамические размеры заголовков;
  - кастомная палитра;
  - глобальные стили через CssBaseline;
  - внедрил переключение темы.

### 5. Страница Practice и Dashboard

Разработал и доработал страницы:

- **Practice:**
  - layout страницы для отображения квизов;
  - состояния загрузки и ошибок;
  - интеграция с API.
- **Dashboard:**
  - графики прогресса (ant-design-charts);
  - скелетоны и обработка ошибок;
  - переход с моковых данных на API;
  - адаптация под локализацию;
  - рефакторинг под единый endpoint API.

### 6. Работа с API и схемами данных

спроектировал:

- схему квиза;
- схему ответов пользователя;
- схему результатов;
- адаптировал фронтенд под API;
- внедрил локализацию в структуру данных;
- работал с .env в Vite для переключения между mock и API режимами.

### 7. Тестирование

Освоил Vitest + Testing Library:

- написал unit-тесты для компонентов QuizNavigation и QuizProgressBar;
- применил подход с функцией setup для уменьшения дублирования кода;
- протестировал:
  - рендер;
  - отображение данных;
  - поведение UI.

### 8. Работа с Git и командной разработкой

- активно участвовал в code review;
- создавал и ревьюил pull request’ы;
- решал конфликты при rebase;
- работал с git workflow (включая ошибки с force push и rebase).

## 9. Исследование LLM-интеграции

Провел исследование по внедрению AI:

- изучил варианты:
  - Ollama;
  - Together AI;
  - Groq API;
- выявил ограничения:
  - невозможность безопасного вызова API с фронта;
  - необходимость backend-прокси.

## Основные сложности

- управление глобальным состоянием и его синхронизация с UI;
- работа с drag-and-drop и коллизиями;
- проектирование универсальных JSON-схем;
- интеграция с API без четкого backend-контракта;
- настройка темизации в MUI;
- разрешение git-конфликтов и работа с rebase.
