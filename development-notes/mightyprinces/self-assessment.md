# Описание работы

В рамках проекта я занималась разработкой интерактивного квиза Code Ordering Widget и страницы Results для отображения результатов всех типов квизов. Параллельно с разработкой с нуля изучала React, Material UI, dnd-kit.

---

## Два личных Feature Component

### Feature Component #1 — Code Ordering Widget

**Complex Component:** Интерактивный квиз, в котором пользователь перетаскивает строки кода в правильном порядке, чтобы собрать корректную функцию. Спроектировала и реализовала компонент полностью с нуля — от дизайна в Figma до интеграции с API бэкенда.

**Что реализовано:**

- Drag & Drop строк кода через библиотеку `dnd-kit` (`useSortable`, `DragDropProvider`, `move`)
- Подсветка синтаксиса кода в каждой строке через `react-syntax-highlighter` с поддержкой светлой и тёмной тем
- Визуальные отступы (indentation) для каждой строки, чтобы код выглядел как в IDE
- Навигация между вопросами с кнопками «Назад» / «Далее» / «Завершить»
- Кнопка перехода к следующему заданию становится активной, только если пользователь действительно изменил позицию элемента
- Сохранение и восстановление ответов при навигации между вопросами через Zustand
- Прогресс-бар с отображением текущего вопроса и процента выполнения
- Формирование payload ответов и отправка на сервер через `submitQuizAnswers`
- Адаптивная вёрстка от 320px — при нехватке ширины текст строк переносится по символам
- 5 unit-тестов (Vitest + Testing Library): рендер, прогресс-бар, заголовок, текст задания
- Подготовка моковых данных: 3 уровня сложности, темы: debounce, throttle, closures, promises, async/await, memoization, curry, deep clone, event emitter, reduce

**Что было сложным и какие проблемы решала:**

- Выбор и освоение dnd-kit — изучала библиотеку с нуля, разбиралась с типизацией событий, хуком `useSortable`, стратегиями коллизий
- Адаптивность: сначала сделала горизонтальный скролл для строк кода (по аналогии с блоками кода на сайтах), но поняла, что скролл конфликтует с drag & drop — элемент таскается вместо прокрутки. Убрала скролл, добавила перенос текста по словам, а затем и по символам для самых узких экранов
- Сохранение порядка строк при навигации назад — нужно было восстанавливать пользовательский порядок из стора, а не исходный с сервера. Реализовала пересортировку массива строк по сохранённому порядку ID
- Интеграция с бэкендом — подготовила данные для базы, согласовала формат с бэкенд-разработчиком, убрала shuffle с фронта (с сервера данные приходят уже перемешанными)

### Feature Component #2 — Страница Results

**Rich UI Screen:** Универсальная страница результатов, отображающая итоги прохождения для всех 5 типов квизов с различным UI для каждого типа.

**Что реализовано:**

- Отображение общей статистики: количество правильных ответов, процент, цветовая индикация (зелёный ≥70%, жёлтый 30–69%, красный <30%)
- Разная вёрстка результата для каждого типа квиза:
  - **Single Choice / True False** — текст вопроса и ответ пользователя
  - **Code Completion** — блок кода с подсказкой и введённый ответ
  - **Async Sorter** — блок кода и последовательность ответа
  - **Code Ordering** — построчный вывод с подсветкой правильности позиции каждой строки (зелёный/красный фон)
- Иконки `CheckCircle` / `Cancel` для визуальной обратной связи по каждому вопросу
- Навигация: кнопки «Попробовать снова» (сброс состояния через Zustand) и «Вернуться к тестам»
- Скелетон загрузки для состояния ожидания данных
- Поддержка локализации (ru/en) через `react-i18next`
- Адаптивная вёрстка от 320px

**Что было сложным и какие проблемы решала:**

- Универсальность компонента — нужно было обработать 5 разных типов квизов с разными структурами данных в одном компоненте. Использовала discriminated unions по полю `quizTask.type` и приведение типов для каждого варианта
- Отображение результатов Code Ordering — нужно было показать, какие строки пользователь расставил правильно, а какие нет. Парсила ID строки для определения корректной позиции и подсвечивала каждую строку зелёным или красным
- Работа с дженериками — компонент `Results<T extends UserAnswerPayload>` использует generic для типизации ответов разных квизов
